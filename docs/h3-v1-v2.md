# H3 v1 vs v2 Usage

This package supports both H3 v1 (1.15.x) and H3 v2 (2.0 beta). The public API you import from this package is the same across versions; the differences are in how you wire handlers and middleware with H3 itself.

## Choosing the correct entry

- H3 v1 (default):
  - Import path: `auth-h3client` or `auth-h3client/v1`
  - Peer dependency: `h3@^1.15.4`

- H3 v2:
  - Import path: `auth-h3client/v2`
  - Peer dependency: `h3@^2.0.0-beta.4`

The root export (`auth-h3client`) resolves to v1 to preserve backwards compatibility. To opt into H3 v2, import from the `v2` subpath.

## Key H3 differences

- Handlers
  - v1: `defineEventHandler(...)`
  - v2: `defineHandler(...)`

- Router middleware
  - v1: chain `router.use('/path', mw)` for each route layer
  - v2: pass `{ middleware: [mw1, mw2] }` as the 3rd argument to `router.METHOD`

- Redirects and errors
  - v1: `sendRedirect(event, url)`, `createError` / `H3Error`, `setResponseStatus`, `appendHeader`
  - v2: `redirect(event, url)`, throw `HTTPError`, set status/headers via the returned `Response` or `event.res`

- Plugins and hooks
  - v1: mutate `app.options.onRequest/onAfterResponse/onError`
  - v2: use `definePlugin`, `onRequest`, `onResponse`, `onError` and register with `app.use(...)`

The exported controllers, middlewares, and route registrars from this package are named the same in both versions (e.g., `useAuthRoutes`, `magicLinksRouter`, `useOAuthRoutes`, `ensureValidCredentials`, etc.).

## Quickstart examples

Below are minimal setups for each H3 version. Both examples assume you have called `configuration({...})` once during boot.

### H3 v1

```ts
import { createApp, createRouter } from 'h3';
import {
  configuration,
  httpLogger,
  isIPValid,
  botDetectorMiddleware,
  generateCsrfCookie,
  useAuthRoutes,
  magicLinksRouter,
  useOAuthRoutes,
} from 'auth-h3client/v1';

configuration(/* your config */);

const app = createApp();
// v1 logger registers via app.options hooks
httpLogger()(app);
app.use(isIPValid);
app.use(botDetectorMiddleware);
app.use(generateCsrfCookie);

const router = createRouter();
useAuthRoutes(router);
magicLinksRouter(router);
useOAuthRoutes(router);

app.use(router);
```

### H3 v2

```ts
import { createApp, createRouter } from 'h3';
import {
  configuration,
  httpLogger,
  isIPValid,
  botDetectorMiddleware,
  generateCsrfCookie,
  useAuthRoutes,
  magicLinksRouter,
  useOAuthRoutes,
} from 'auth-h3client/v2';

configuration(/* your config */);

const app = createApp();
// v2 logger is a plugin: register directly
app.use(httpLogger);
app.use(isIPValid);
app.use(botDetectorMiddleware);
app.use(generateCsrfCookie);

const router = createRouter();
useAuthRoutes(router);
magicLinksRouter(router);
useOAuthRoutes(router);

app.use(router);
```

## Route-level middleware example

Registering `login` with CSRF, content-type, and size limits differs slightly.

- H3 v1

```ts
router.use('/login', verifyCsrfCookie);
router.use('/login', contentType('application/json'));
router.use('/login', limitBytes(1024));
router.post('/login', loginHandler);
```

- H3 v2

```ts
router.post('/login', loginHandler, {
  middleware: [verifyCsrfCookie, contentType('application/json'), limitBytes(1024)]
});
```

## Authoring custom handlers

When writing your own handlers alongside the ones provided here:

- H3 v1

```ts
import { defineEventHandler } from 'h3';

router.get('/protected', defineEventHandler(async (event) => {
  const mfa = await ensureValidCredentials(event);
  if (mfa && 'text' in mfa) {
    event.res.statusCode = 202;
    return mfa;
  }
  return { ok: true };
}));
```

- H3 v2

```ts
import { defineHandler } from 'h3';

router.get('/protected', defineHandler(async (event) => {
  const mfa = await ensureValidCredentials(event);
  if (mfa && 'text' in mfa) {
    event.res.status = 202;
    return mfa;
  }
  return { ok: true };
}));
```
